## Project Overview

Civic Health Alliance is an interactive data visualization that maps the relationship between civic engagement and health outcomes across the US, drilling from national to individual level. Built as a single-page React app with SVG-based maps and animations.

## Commands

```bash
npm run lint     # ESLint (flat config, v9)
npm run preview  # Preview production build
```

No test framework is configured.

## Architecture

### Monolithic Single-File Structure

Currently, nearly all application code lives in `src/App.jsx` (~730 lines). This includes data constants, color utilities, all UI components, and the main App component. The `.planning/ROADMAP.md` outlines a plan to decompose this.

### Zoom Level Hierarchy

The app is organized around a 4-level drill-down controlled by the `level` state variable:

- **Level -1**: Title/landing screen
- **Level 0**: US hexagonal state map (50 states, colored by life expectancy + voter turnout)
- **Level 1**: NYC community district map (59 districts, SVG paths in `CD_PATHS`)
- **Level 2**: Hospital/health center scene (departments with person figures)
- **Level 3**: Individual person story (timeline narrative)

Navigation between levels uses a `zoomTo(nextLevel, personId?)` callback with a two-phase animation: fade out (400ms) → update state → fade in (450ms).

### Data Layer

All data is embedded as JavaScript object literals at the top of `App.jsx`:
- `STATE_DATA` — 50 states with life expectancy, voter turnout, uninsured rate
- `HEX_GRID` — `{stateCode: [col, row]}` hex grid positions
- `CD_PATHS` — ~80KB of SVG path strings for NYC community districts
- `NYC_DISTRICTS` — district metadata (name, life expectancy, voter turnout)
- `HOSPITAL_PEOPLE` — people in the hospital scene
- `DEPARTMENTS` — hospital department layout

### Styling Approach

- **Inline styles exclusively** — no CSS classes on components, all `style={{}}` objects
- Color constants: `RED="#B8120E"`, `CREAM="#FAF8F4"`
- Google Fonts loaded via `<style>` tag: Libre Franklin, JetBrains Mono, Playfair Display
- Animations use a mix of CSS `@keyframes` and SVG `<animate>` SMIL elements

### Color Mapping Functions

- `leColor(le)` — life expectancy → 4-tier color scale (red → yellow → light green → dark green)
- `engagementColor(t)` — voter turnout % → engagement color gradient
- `typeColor` object — maps data categories to colors

### Component Patterns

- All function components with hooks (no class components)
- Key components: `Pulse`, `USMap`, `NYCMap`, `PersonFig`, `HospitalScene`, `PersonStory`, `InfoPanel`, `NavBar`
- State managed via `useState` hooks in App: `level`, `phase`, `hovState`, `hovHood`, `hovPerson`, `hovDept`, `selectedPerson`
- Defensive null checks (`if(!person) return null`) rather than error boundaries

## Pattern Overview
- All application code (730 lines, 72KB) lives in one file: `src/App.jsx`
- No routing library; navigation is a state-driven zoom-level system
- Data is embedded as inline JavaScript object literals (no API, no database)
- All styling is inline (`style={{}}` objects); no CSS modules or utility classes
- SVG-based data visualization rendered entirely in React (no D3, no charting library)
- Zero external dependencies beyond React and ReactDOM
