## Spacing Scale

Phase 3 makes no changes to spacing in HTML elements. All layout work is in SVG coordinate space, which uses its own unit system (SVG user units, viewBox `0 0 100 96`). The standard spacing scale is inherited and unchanged.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**Exceptions:**
- Touch targets: minimum 44px on all interactive SVG person figures (HOSP-03). Achieved via a 12├Ś12 SVG unit invisible `<rect fill="transparent">` which at the minimum supported viewport (768px, 4.08 px/SVG unit) produces 48.96px ŌĆö above the 44px floor. This is an SVG-domain measurement, not an HTML spacing token.

**Source:** REQUIREMENTS.md HOSP-03; RESEARCH.md Pattern 4 (touch target sizing).

### Invisible Touch Target Rect (HOSP-03)

One `<rect fill="transparent">` is added per person, inside their `<g>` element, before `PersonFig` and `Pulse`. This ensures the rect draws beneath but receives pointer events across the full 12├Ś12 area.

| Property | Value | Rationale |
|----------|-------|-----------|
| x | `p.x ŌłÆ 6` | Centers the 12-unit rect on the figure anchor |
| y | `p.y ŌłÆ 2` | Covers head top (p.y ŌłÆ 1.35) with 0.65 unit margin above |
| width | 12 SVG units | 12 ├Ś 4.08 = 48.96px at minimum viewport ŌĆö exceeds 44px |
| height | 12 SVG units | 12 ├Ś 4.08 = 48.96px at minimum viewport ŌĆö exceeds 44px |
| fill | `"transparent"` | Visually invisible; participates in SVG hit testing |
| style | `{cursor:"pointer"}` | Preserves pointer cursor on touch target area |

**Touch target non-overlap verification:** Closest figure pair is patient-one (x=72) and dr-saffo (x=29). Their touch rects span x=66..78 and x=23..35 respectively. No overlap.

## Interaction Contract

### HOSP-01: Figure Centering

- **What changes:** `x` values for `patient-one` (60 ŌåÆ 72) and `patient-two` (72 ŌåÆ 60) in `HOSPITAL_PEOPLE` data constant.
- **What does not change:** `PersonFig` component, `DEPARTMENTS` layout, room geometry, SVG viewBox.
- **Verification:** Each figure's x must match its room's horizontal center (room.x + room.w/2).

### HOSP-02: Text Clearance

- **What changes:** `y` values for all four people: upper rooms 36 ŌåÆ 38, lower rooms 69 ŌåÆ 71.
- **What does not change:** Department text positions (label, metric, metricLabel, note) ŌĆö these are correct; only figures move.
- **Anti-pattern:** Do not move text to fix overlap. Move figures only.
- **Verification:** After correction, 2.65 SVG units clearance between metricLabel baseline and figure head top.

### HOSP-03: Touch Targets

- **What changes:** Add one `<rect fill="transparent" x={p.x-6} y={p.y-2} width={12} height={12} style={{cursor:"pointer"}}/>` inside each person `<g>` in `HospitalScene`, placed before `PersonFig` and `Pulse` in render order.
- **Event handlers:** The `<rect>` inherits pointer events from its parent `<g>`. The `<g>` must carry `onMouseEnter`, `onMouseLeave`, `onTouchEnd`, `onTouchCancel`, and `onClick` handlers.
- **Phase 2 dependency:** If Phase 2 has been executed before Phase 3, `onTouchEnd` and `onTouchCancel` on the person `<g>` elements may already be present. The executor must check and not duplicate. If absent, add them ŌĆö both `onTouchEnd` and `onTouchCancel` must call `onHoverPerson(null)` to clear sticky hover state on touch screens.
- **Instruction update (deferred):** The existing hover instruction "Hover rooms for context ┬Ę Click any person to read their story" at line 575 is mouse-centric. This is out of scope for Phase 3 per REQUIREMENTS.md ŌĆö no content changes in this phase.

### Room Hover (No Change)

Room `<g>` elements in `HospitalScene` have their own `onMouseEnter`/`onMouseLeave` for `hovDept`. If Phase 2 has not yet added `onTouchEnd`/`onTouchCancel` to these room `<g>` elements, Phase 3 must add them (same pattern as person figures: call `onHoverDept(null)` on touch end/cancel).
