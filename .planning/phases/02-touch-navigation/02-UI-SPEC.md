## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline button padding supplement |
| sm | 8px | Gap between breadcrumb chevron and label |
| md | 16px | NavBar internal padding, header gaps |
| lg | 24px | Section padding in PersonStory |
| xl | 32px | Layout gaps between panels |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (not used in NavBar) |

**Exceptions:**
- Touch targets: minimum 44px height/width on all interactive elements (NAV-01, NAV-02). This is a hard floor enforced via `minHeight:44, minWidth:44` on buttons. 44 is not on the 8-point scale but is the iOS HIG / Google Material / WCAG 2.5.8 touch target minimum.
- NavBar height: 61px (existing, unchanged) — legacy value from initial build; no change required. Exceeds the 44px minimum.
- MapBtn height: 45px (existing, unchanged) — legacy value from initial build; meets 44px touch minimum.
- LeftPanel Back button: currently `padding:"10px 20px"` + `fontSize:12` gives ~38px computed height. Must be upgraded to `padding:"14px 20px"` to reach 46px minimum, OR add `minHeight:44`.

**Source:** REQUIREMENTS.md NAV-01 "44px+ touch target"; RESEARCH.md Architecture Patterns NAV-01 section.

## Interaction Contract

This section is specific to Phase 2's touch-first interaction requirements and supplements the standard contract sections above.

### NAV-01: Back Arrow Button in NavBar

- **Visibility:** Rendered when `level > 0`. Hidden at `level <= 0` (title screen has no back target).
- **Position:** Leftmost interactive element in NavBar, before the brand wordmark. Alternatively, between the brand and the vertical divider — executor may choose; both placements satisfy the requirement.
- **Touch target:** `minWidth:44, minHeight:44` enforced via inline style.
- **Glyph:** Unicode left arrow `←` (&#8592;), `fontSize:20`, `fontWeight:700`.
- **Color:** `#f5f5f5` (matches existing NavBar button text).
- **Background:** `none` (transparent — matches existing NavBar ghost button pattern).
- **Action:** Calls `onGoLevel(level - 1)` (or `onBack` prop — see prop threading note below).
- **Animation guard:** The App-level `phase !== "idle"` guard on `zoomTo` prevents double-triggering; the button itself does not need a disabled state.
- **Source:** REQUIREMENTS.md NAV-01; RESEARCH.md NAV-01 code example.

### NAV-02: Tappable Breadcrumb Segments

- **Current state:** "Health Center" and "Story" are `<span>` elements — non-interactive.
- **Change:** At `level===3`, convert "Health Center" breadcrumb span to a `<button>`. "Story" remains a non-interactive `<span>` (current level — no self-navigation).
- **Touch target:** `minHeight:44` on the Health Center button.
- **Guard:** Only navigates backward. `onClick={() => zoomTo(2, null)}`. At `level===2`, "Health Center" is the current level and remains a `<span>` (no button needed there).
- **Existing tabs:** "United States Map" and "New York City Map" are already `<button>` elements with `onGoUS` / `onGoNYC` props. These satisfy backward navigation at those levels already.
- **Style parity:** Health Center button uses identical style to existing `<span>` text (Libre Franklin, 13px, italic, 600 weight, `#f5f5f5`) with `background:none`, `border:none`, `cursor:pointer` added.
- **Source:** REQUIREMENTS.md NAV-02; RESEARCH.md NAV-02 code example.

### NAV-03: Eliminate 300ms Tap Delay

- **Mechanism:** `touchAction:"manipulation"` inline style applied to the App root `<div>` (line 723 in App.jsx). Cascades to all children.
- **Also apply to:** Every new `<button>` added in this phase (NavBar back arrow, Health Center breadcrumb button) as a belt-and-suspenders measure, since the root already covers them.
- **No JavaScript changes required** for this requirement alone.
- **Source:** REQUIREMENTS.md NAV-03; RESEARCH.md NAV-03 architecture pattern.

### NAV-04: Swipe Left/Right Navigation

- **Gesture direction:** Swipe right = back (level - 1). Swipe left = forward (level + 1, levels 0 and 1 only).
- **Threshold:** 50px horizontal delta.
- **Horizontal guard:** Only trigger if `Math.abs(deltaX) > Math.abs(deltaY)` — prevents vertical LeftPanel scrolling from triggering swipe navigation.
- **Animation guard:** If `phase !== "idle"`, swipe is a no-op.
- **Level constraints:**
  - Swipe right at `level <= 0`: no-op.
  - Swipe left at `level === 2`: no-op (cannot navigate to level 3 without an explicit person tap).
  - Swipe left at `level === 3`: no-op (deepest level).
  - Swipe left at `level === -1` (title screen): no-op.
- **Implementation hooks:** `useRef` (touchStartX, touchStartY) + `useCallback` (handleTouchStart, handleTouchEnd). The `useRef` import must be added to the React import line.
- **Source:** REQUIREMENTS.md NAV-04; RESEARCH.md NAV-04 code example + Pitfall 3 (Y-delta guard).

### Sticky Hover Fix (Implicit to Phase 2)

This is not a named requirement but is documented as a blocker in STATE.md and is a direct consequence of touch optimization work.

- **Affected components:** `USMap` `<g>` elements (onMouseLeave), `NYCMap` `<g>` elements (onMouseLeave), `HospitalScene` room and person `<g>` elements (onMouseLeave).
- **Fix:** Add `onTouchEnd` and `onTouchCancel` handlers calling the same null-clear function as the existing `onMouseLeave`.
- **Source:** STATE.md Blockers/Concerns; RESEARCH.md Sticky Hover Fix section.

### LeftPanel Back/Next Buttons

- **Back button (`level > 0`):** Current `padding:"10px 20px"` gives ~38px height. Must add `minHeight:44` inline style to meet NAV-01 touch target minimum. Style is otherwise unchanged.
- **Next button (`level < 2`):** Current `padding:"10px 20px"` gives ~38px height. Add `minHeight:44` for consistency even though NAV-01 targets the back arrow specifically.
- **Note:** LeftPanel is hidden at level 3; the NavBar back arrow provides the sole back navigation path from the story view.
