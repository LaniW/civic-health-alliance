# Phase 4 — UI Design Contract

> Visual and interaction contract for Phase 4: Real Content and Data Verification.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase character:** This is a data-correctness and content phase, not a new-feature phase.
> No new components, no new layout patterns. The design contract enforces continuity with
> the existing system and specifies the exact visual consequences of the data changes
> (corrected voter turnout values shift color tier distribution; citation link text must
> update to match new sources).

must NOT be adjusted to preserve the old appearance. The corrected data is more accurate;
the colors should reflect that. Document this as intentional in the commit message.

## Copywriting Contract

Phase 4 touches two categories of user-visible text: (1) source citation links in the
LeftPanel legends, and (2) patient story content in HOSPITAL_PEOPLE (consent-gated).

### Source Citation Links (DATA-01, DATA-03)

These appear in the Level 0 legend card (App.jsx lines 341–344) and Level 1 legend card
(App.jsx lines 395–397). The current link text and URLs are wrong (reference MAP 2024 VT
source instead of UF Election Lab VEP source). The correct values are:

| Element | Current (Wrong) | Correct |
|---------|----------------|---------|
| Level 0 VT citation link text | "Voter Turnout — MAP, 2024" | "Voter Turnout — UF Election Lab, 2020 VEP" |
| Level 0 VT citation href | MAP URL | `https://election.lab.ufl.edu/voter-turnout/2020-general-election-turnout/` |
| Level 0 uninsured citation text | "Uninsured Rate — KFF, 2024" | "Uninsured Rate — KFF ACS, 2022–2023" |
| Level 0 LE citation text | "Life Expectancy — CDC NVSR, 2023" | "Life Expectancy — CDC NVSR Vol. 74 No. 12, 2022" |
| Level 1 VT citation text | "Voter Turnout — NYC Equity, 2024" | "Voter Turnout — NYC CFB Community Profiles, 2020" |
| Level 1 LE citation text | "Life Expectancy — NYC DOHMH, 2023" | "Life Expectancy — NYC DOHMH CHP 2021 (2010–2019 avg)" |

### Patient Story Content (STORY-02)

**Consent gate:** All story text changes are gated on human confirmation from Stella Saffo.
Two paths diverge at consent confirmation:

**Path A — Consent confirmed for kiosk display:**
- Story text publishes as-is (minor copy refinements only)
- Dr. Saffo direct quotes remain attributed
- No copy changes required beyond what is already in HOSPITAL_PEOPLE

**Path B — Consent limited (not approved for public kiosk display):**
- Remove direct quote attribution from patient-one timeline entry:
  - Current: `"As Dr. Saffo said: 'It wasn't HIV that killed him. It ended up being an insurance issue that killed him.'"`
  - Correct: `"It wasn't HIV that killed him. The Medicaid benefit limit — a policy decision — ended his life."`
- Remove direct quote from dr-saffo timeline entry:
  - Current: `"'It ended up being an insurance issue that killed him.'"`
  - Correct: Paraphrase without quotation marks or attribution
- Remove direct attribution from patient-three timeline entry:
  - Current: `"'Lucky for her, New York State offers some good benefits,' Dr. Saffo said."`
  - Correct: `"New York State offers benefits tied to HIV status and pregnancy — a policy difference that determined whether she had a home."`
- All four figures remain in HOSPITAL_PEOPLE with generic descriptive names (no patient names to remove — already compliant)
- Reframe stories as "composite cases illustrating systemic patterns" if needed

### Primary CTA

| Level | CTA Label | Location |
|-------|-----------|----------|
| Level -1 (title) | "BEGIN" | Title screen button |
| Level 0 | "Next →" | LeftPanel nav button (unchanged) |
| Level 1 | "Next →" | LeftPanel nav button (unchanged) |
| Level 2 | none (click person figures) | Instruction text at SVG bottom |

Phase 4 does not introduce new CTAs.

### Empty State

No empty states are added in Phase 4. All data is inline; the visualization renders
completely regardless of data values.

### Error State

No error states are added in Phase 4. No async operations; no user input validation.
If a NYC district LE or VT value is audited and found to be genuinely unavailable in
the primary source, use the notation `le: null` and guard the display with the existing
null-check pattern. Do not display "N/A" or "–" without implementing the null guard.

### Destructive Actions

None in Phase 4. No user-initiated data deletion, reset, or irreversible action.
