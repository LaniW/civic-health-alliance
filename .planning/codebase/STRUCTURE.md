## Directory Purposes

**src/:**
- Purpose: All JavaScript/JSX application code
- Contains: React components, data constants, styling
- Key files: `App.jsx` (567 lines — monolithic component)

**public/:**
- Purpose: Static files copied directly to build output
- Contains: Images, icons, favicon assets
- Behavior: Contents served at root path during build

**.planning/codebase/:**
- Purpose: Architecture and code mapping documentation
- Generated by: GSD mapping tools
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, STACK.md, INTEGRATIONS.md, CONCERNS.md

## Key File Locations

**Entry Points:**

- `index.html`: HTML document that loads React app; contains single `<div id="root">` mount point
- `src/main.jsx`: JavaScript entry point; imports App component and mounts to DOM
- `src/App.jsx`: React component tree root; contains all application logic

**Configuration:**

- `vite.config.js`: Build tool configuration; enables React plugin via `@vitejs/plugin-react`
- `eslint.config.js`: Linting rules for code quality
- `package.json`: Project metadata, dependencies (React 19.2.0, Vite 7.3.1), and npm scripts

**Core Logic:**

- `src/App.jsx`: Single monolithic file containing:
  - Data constants (lines 7–248): STATE_DATA, HEX_GRID, CD_PATHS, NYC_DISTRICTS, HOSPITAL_PEOPLE, DEPARTMENTS
  - Color mapping functions (lines 194–250): leColor, engagementColor, typeColor
  - Components (lines 252–483): Pulse, USMap, NYCMap, PersonFig, HospitalScene, PersonStory, InfoPanel, NarrativeOverlay
  - Main App component (lines 488–567): State management and layout orchestration

**Styling:**

- `src/App.css`: Legacy unused styles (542 bytes)
- `src/index.css`: Base document styles (68 lines)
- Inline styles: All component styling via `style={{}}` props and `<style>` tag in App root

**Assets:**

- `src/assets/`: Directory for images (currently empty or minimal)
- `public/`: Favicon and logo files (vite.svg assumed)

## Naming Conventions

**Files:**

- `.jsx` extension: All JavaScript files with JSX
- Uppercase first letter: `App.jsx`, `main.jsx` (entry points and main component)
- Lowercase `.css`: Stylesheets
- `.config.js`: Configuration files (`vite.config.js`, `eslint.config.js`)
- `.json`: Manifest files (`package.json`, `package-lock.json`)

**Directories:**

- Lowercase singular: `src`, `public`, `assets`
- Plural for collections: `assets`
- Dot-prefixed for hidden: `.planning`, `.git`, `.gitignore`

**Variables and Constants:**

- SCREAMING_SNAKE_CASE: Large data objects (STATE_DATA, HEX_GRID, CD_PATHS, NYC_DISTRICTS, HOSPITAL_PEOPLE, DEPARTMENTS)
- camelCase: Functions (leColor, engagementColor, zoomTo) and variables (hovState, selectedPerson, phase)
- PascalCase: React components (USMap, NYCMap, PersonFig, HospitalScene, PersonStory, InfoPanel, NarrativeOverlay, Pulse)

**React Components:**

All are function components using hooks (useState, useCallback).

## Where to Add New Code

**New Feature (e.g., filtering, search):**
- Primary code: `src/App.jsx` (add state variables, helper functions, conditional rendering)
- Tests: Not configured; would go in `src/App.test.jsx`
- No separate component files exist; add inline or refactor into new file

**New Interactive Level (e.g., zoom level 4):**
- Add data structure to top of `src/App.jsx` (e.g., `LEVEL_4_DATA = {...}`)
- Create rendering component function (follow pattern of USMap, NYCMap, HospitalScene)
- Update `level` state enum (currently -1, 0, 1, 2, 3)
- Add conditional rendering block to main App (follow pattern at lines 546–556)
- Update breadcrumb array `crumbs` calculation (line 508–509)

**New Visualization Component:**
- Location: `src/App.jsx` (insert before App export, follow naming convention)
- Pattern: Function component accepting props for data, handlers (onHover, onClick)
- SVG-based: Use `<svg viewBox="...">` structure with nested elements
- Styling: Inline via style objects and transform attributes

**Utilities (color mapping, calculations):**
- Location: Top of `src/App.jsx` alongside leColor, engagementColor (lines 194–250)
- Pattern: Pure functions with single responsibility
- Export: Not needed—referenced directly in same file

**Static Data:**
- Location: Top of `src/App.jsx` as CONSTANT_NAME (lines 7–248)
- Format: JavaScript object literals or arrays
- Shape: Define consistent structure for scalability

**Styling:**
- Global styles: Add to `<style>` tag in App component (lines 519–524)
- Component styles: Inline via `style={{}}` props
- CSS classes: Avoid; not used in codebase

## Monolithic File Architecture Note

**Important:** The entire application lives in `src/App.jsx` (567 lines). There is no component separation or modular file structure. This means:

- All state management is in the root App component
- All components are function declarations in the same file
- All data constants are at file top
- All styling is inline or in root `<style>` tag
- Adding features requires editing this single file
- Component reuse is impossible (no exports except default App)

**For refactoring:** To split into multiple files, consider:
- `src/components/`: Create separate files (USMap.jsx, NYCMap.jsx, etc.)
- `src/data/`: Create separate data files (stateData.js, nycDistricts.js, etc.)
- `src/utils/`: Create utility files (colorFunctions.js, animations.js, etc.)
- But this would require import/export refactoring
